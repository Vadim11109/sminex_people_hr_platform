## Маршрутизация

### Публичные маршруты
| Маршрут | Описание |
|---------|----------|
| `/auth/signin` | Страница входа через Microsoft |
| `/auth/error` | Ошибка авторизации |

### HR (`/hr/*`)
| Маршрут | Описание |
|---------|----------|
| `/hr` | Дашборд — статистика, активность |
| `/hr/cycles` | Управление циклами оценки |
| `/hr/users` | Управление сотрудниками и ролями |
| `/hr/analytics` | Аналитика по грейдам и командам |
| `/hr/grades` | Система грейдов по должностям |
| `/hr/templates` | Шаблоны ассесмента |

### Руководитель (`/manager/*`)
| Маршрут | Описание |
|---------|----------|
| `/manager` | Моя команда — статус ассесмента |
| `/manager/pending` | Сотрудники, ожидающие оценки |
| `/manager/results` | Результаты команды |
| `/manager/assess/[userId]` | Форма оценки сотрудника |
| `/manager/gap/[userId]` | Gap-анализ сотрудника |

### Сотрудник (`/employee/*`)
| Маршрут | Описание |
|---------|----------|
| `/employee` | Мои ассесменты — активные и завершённые |
| `/employee/history` | История грейдов по циклам |
| `/employee/assessment/[id]` | Форма само-оценки |

## Аутентификация и авторизация

### Поток авторизации (production)
```
Пользователь → /auth/signin
→ Microsoft Azure AD (OAuth 2.0 / OIDC)
→ Callback → NextAuth создаёт/обновляет User в БД
→ Редирект на / → определяется роль → /hr, /manager или /employee
```

### Dev-режим
В `src/proxy.ts` middleware отключён — все маршруты открыты без авторизации.  
В каждом layout-файле подставляется mock-сессия:
```ts
const mockSession = { user: { id: 'dev', name: 'Dev HR', email: 'hr@sminex.ru', roles: ['HR'] } }
```

### Определение роли после входа
Роли НЕ берутся из Azure AD. Они назначаются HR вручную в `/hr/users`.  
При первом входе пользователь получает роль `EMPLOYEE` по умолчанию.

## Компоненты

### `Sidebar`
- Принимает: `user`, `role`, `navItems[]`
- Показывает: лого SMINEX | People, навигацию, аватар, кнопку выхода
- При нескольких ролях показывает кнопку «Сменить режим» → `/select-role`

### `GlobalNav`
- Плавающая панель в правом нижнем углу
- Только для разработчиков/администраторов
- Позволяет быстро переходить между всеми разделами платформы
- В production должна быть скрыта или удалена

## Стили

Все стили — CSS-переменные в `src/app/globals.css`.  
Ни Tailwind, ни UI-библиотеки не используются намеренно.

Ключевые переменные:
```css
--purple        /* акцент сотрудника */
--blue          /* акцент менеджера / действия */
--green         /* акцент HR / успех */
--amber         /* предупреждения */
--surface       /* фон карточек */
--border        /* границы */
--text          /* основной текст */
--muted         /* второстепенный текст */
--hint          /* подсказки */
```
